Practical 1: Amazon Web Services EC2 Case Study

What is EC2?

EC2 stands for Elastic Compute Cloud.
It is a virtual server provided by AWS in the cloud.

Instead of buying a physical computer/server, we can rent a virtual machine from AWS.

Main Concepts

EC2 = Virtual machine in cloud

On-demand instance = Pay only when using

AMI = Amazon Machine Image (OS template)

VPC = Virtual Private Cloud

Subnet = Small network inside VPC

Elastic IP = Static public IP


Steps Performed

1. Login into AWS


2. Open EC2 Dashboard


3. Select region


4. Launch instance


5. Choose AMI


6. Select instance type (t2.micro)


7. Configure VPC and subnet


8. Assign IP


9. Launch EC2 instance



Why EC2 is used?

Hosting websites

Cloud applications

Testing servers

Scalable computing


Advantages

Cost saving

Scalable

Easy deployment

No physical hardware needed

Answer: IaaS (Infrastructure as a Service)

Practical 3: Salesforce Application using Apex

What is Salesforce?

Salesforce is a cloud-based CRM platform.

CRM = Customer Relationship Management.

It helps companies manage customers, sales, and support.

What is Apex?

Apex is a programming language developed by Salesforce.

It is:

Object-oriented

Strongly typed

Java-like language


Features of Apex

Supports DML operations

SOQL/SOSL queries

Easy testing

Multitenant support

Automatic upgrades


DML Operations

Insert

Update

Delete

Upsert


SOQL

Salesforce Object Query Language used to fetch data.

Why Apex is Used?

For complex business logic which cannot be done using normal Salesforce tools.

Practical 5: Private Cloud using VMware ESXi + HDFS

Aim

To setup a private cloud over LAN.

What is Private Cloud?

A cloud infrastructure used only within an organization.

What is VMware ESXi?

A hypervisor used to create and manage virtual machines.

What is HDFS?

Hadoop Distributed File System.

Used to store files in blocks across multiple systems.

Main Operations

Divide file into blocks

Upload/download files

Encryption

Cloud controller setup


Requirements

64-bit processor

Minimum 4 GB RAM

Virtualization enabled

Network adapters


Steps Performed

1. Install ESXi


2. Configure management network


3. Set IPv4 and DNS


4. Create datacenter


5. Create cluster


6. Add hosts


7. Configure failover and DRS



Important Terms

Hypervisor

Software that creates virtual machines.

DRS

Distributed Resource Scheduler.

Balances VM load automatically.

Failover

Backup system if one host fails.
